# Collections and Arrays

AutoParams can generate collections and arrays automatically. By default, the size of generated collections and arrays is 3, but you can override this using annotations such as [`@Size`](https://javadoc.io/static/jakarta.validation/jakarta.validation-api/3.0.2/jakarta/validation/constraints/Size.html).

For example, the following tests verify that AutoParams generates an `ArrayList<String>` and a `String[]` with exactly 5 elements when the `@Size(min = 5)` constraint is applied:

<JavaKotlinCodeBlock>
```java
@Test
@AutoParams
void testMethod(@Size(min = 5) ArrayList<String> arrayList) {
    assertThat(arrayList).hasSize(5);
}
```

```kotlin
@Test
@AutoKotlinParams
fun testMethod(@Size(min = 5) arrayList: ArrayList<String>) {
    assertThat(arrayList).hasSize(5)
}
```
</JavaKotlinCodeBlock>

<JavaKotlinCodeBlock>
```java
@Test
@AutoParams
void testMethod(@Size(min = 5) String[] array) {
    assertThat(array).hasSize(5);
}
```

```kotlin
@Test
@AutoKotlinParams
fun testMethod(@Size(min = 5) array: Array<String>) {
    assertThat(array).hasSize(5)
}
```
</JavaKotlinCodeBlock>

This allows you to work with realistic data sizes while keeping your test code clean and concise.
