# `@Freeze` 애너테이션

`@Freeze` 애너테이션을 사용하면 생성된 값을 "동결"하여 단일 테스트 메서드 내에서 동일한 유형의 다른 매개변수에서 전반에 걸쳐 일관되게 재사용할 수 있습니다.

이를 적용하려면 원하는 메소드 매개변수에 `@Freeze` 애너테이션을 붙이면 됩니다. 그러면 AutoParams가 해당 동결된 값을 복합 개체의 중첩된 필드를 포함하여 동일한 유형의 다른 모든 매개변수에 전파합니다.

<JavaKotlinCodeBlock>
```java
@AllArgsConstructor
@Getter
public class StringContainer {

    private final String value;
}
```

```kotlin
class StringContainer(val value: String)
```
</JavaKotlinCodeBlock>

<JavaKotlinCodeBlock>
```java
@Test
@AutoParams
void testMethodFreeze(String s1, @Freeze String s2, String s3,
                      StringContainer container) {
    assertNotEquals(s1, s2);
    assertEquals(s2, s3);
    assertEquals(s2, container.getValue());
}
```

```kotlin
@Test
@AutoParams
fun testMethodFreeze(s1: String, @Freeze s2: String, s3: String,
                     container: StringContainer) {
    Assertions.assertNotEquals(s1, s2)
    assertEquals(s2, s3)
    assertEquals(s2, container.value)
}
```
</JavaKotlinCodeBlock>

이 예에서 `s2`는 동결되어 동일한 문자열 값이 `s3`와 `container` 내부의 `value` 필드에 재사용됩니다. 한편 `s1`은 독립적으로 생성됩니다.

이와 같이 테스트의 특정 종속성을 일관되되록 유지하면서 나머지 테스트 데이터에 변형을 허용해야 하는 경우 `@Freeze`가 특히 유용합니다.
